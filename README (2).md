## 📖 Theory
Logs are records generated by operating systems, servers, applications, and security devices. Security analysts examine these logs to detect suspicious behavior such as brute-force attacks, unauthorized logins, malware execution, and application failures. Python simplifies this by reading files, matching patterns, extracting IP addresses via regex, and auto-generating reports.

## ⚙️ How It Works
The script (`log_analyzer.py`):
1. Prompts for a log file path and reads its contents.
2. Scans each line for keywords — `error`, `warning`, `critical`, `failed`, `accepted`/`successful`, `sudo`, `ssh` — and tallies counts.
3. Uses a regex pattern (`(?:\d{1,3}\.){3}\d{1,3}`) to extract every IP address, tracking overall frequency and failed-login frequency per IP.
4. Flags any IP with **5 or more failed login attempts** as a possible brute-force source.
5. Prints a summary report to the terminal and saves it as:
   - `analysis_report.txt`
   - `analysis_report.csv`
   - `analysis_report.json`
